📦 Plugin Package Configuration

Each plugin ships as a scoped npm package with @squad/sdk peer dependency

jira-integration/package.json
{ "name": "@squad/plugin-jira-integration", "version": "1.0.0", "main": "dist/index.js", "types": "dist/index.d.ts", "peerDependencies": { "@squad/sdk": "^1.0.0" }, "dependencies": { "axios": "^1.7.9" } }
db-migration-checker/package.json
{ "name": "@squad/plugin-db-migration-checker", "version": "1.0.0", "main": "dist/index.js", "types": "dist/index.d.ts", "peerDependencies": { "@squad/sdk": "^1.0.0" } // zero runtime deps — uses only Node fs/path }

🔗 Test Project Links Plugins Locally

In test-project/package.json, each plugin is linked via file: protocol:

"@squad/plugin-jira-integration": "file:../plugins/jira-integration"
"@squad/plugin-db-migration-checker": "file:../plugins/db-migration-checker"
"@squad/plugin-changelog-generator": "file:../plugins/changelog-generator"
